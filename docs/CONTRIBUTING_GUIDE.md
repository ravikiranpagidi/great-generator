# Contributing Guide

See the root `CONTRIBUTING.md` for the full contributor workflow.

Good places to start:

- add realistic values for a domain pack
- add a notebook or runnable example
- improve API reference docs
- add Spark benchmark coverage
- add tests for a new anomaly type
- add a new industry domain pack

Useful labels for issues:

- `good first issue`
- `documentation`
- `enhancement`
- `bug`
- `domain-pack`
- `realistic-values`
- `spark`
- `pandas`
- `testing`
